Designing a Store Base

In this project, a store base was designed with preferred entries as store items, while employing the use of select statements to order the items by price, showing at least one statistic about the items.

SQL Data Analysis MySQL MYSQL Server

+ Mall Database Setup and Basic Analytics +

Objective

This SQL project involves creating a simple database for a fictional shopping mall, inserting item data, and performing basic statistical queries. It includes commands to initialize the database, create tables, insert records, and retrieve useful metrics.

Step 1: Create the Database

-- Create a new database
CREATE DATABASE mall;

This command initializes a new database named mall for storing structured data about products sold in the mall.

Step 2: Select the Database

-- Select the database to work with
USE mall;

The USE command sets mall as the active database context for upcoming operations.

Step 3: Create the Product Table

CREATE TABLE tospin (
  id INTEGER PRIMARY KEY,
  name TEXT,
  price FLOAT,
  quantity INTEGER,
  sales INTEGER
);

This command creates a table called tospin to store products with attributes like ID, name, price, available quantity, and number of sales.

Step 4: Insert Data

INSERT INTO tospin VALUES (1, "Avatar Game Box", 2000, 40, 30);
INSERT INTO tospin VALUES (2, "handbags", 200, 40, 70);
INSERT INTO tospin VALUES (3, "Detergent", 100, 50, 50);
-- ... (and so on up to 25 rows)
INSERT INTO tospin VALUES (25, "Pencil Case", 10, 100, 900);

A total of 25 products are added to the tospin table, covering a mix of household goods, tech items, and stationery.

Step 5: Perform Analytics

Maximum Price

SELECT max(price) FROM tospin;

Returns the highest priced item in the store.

Minimum Price

SELECT min(price) FROM tospin;

Returns the lowest priced item in the store.

Average Price

SELECT avg(price) FROM tospin;

Calculates the average price of all listed products.

Total Quantity in Stock

SELECT sum(quantity) FROM tospin;

Gives the total count of all item units available in the mall.

Number of Named Products

SELECT count(name) FROM tospin;

Returns the number of products that have a non-null name in the table.

Conclusion

This project demonstrated how to use basic SQL commands to build a database from scratch, populate it with structured data, and generate simple insights using aggregate functions like MAX(), MIN(), AVG(), SUM(), and COUNT().

Data Dig Project On Winston's Donut Logs

This is a project that creates a database and a table in MySQL to store logs of donuts eaten by a person named Winston throughout his life. The purpose of this project is to practice creating a database, defining a table schema, and inserting data into it using SQL commands.

Python Algorithms Machine Learning Data Science

+ DataDig: Donut Consumption Analysis with MySQL +

Project Objective

DataDig is a fun yet technically rich SQL project focused on database creation, schema design, and structured data insertion. As a data analyst, this project showcases your ability to set up a robust data foundation from scratch using MySQL. Through the lens of Winston's lifelong donut-eating habits, the project demonstrates how to build a query-ready dataset with real-world structure and narrative.

Project Summary

This project involves:

  • Creating a new MySQL database called DataDig
  • Designing and implementing a structured table: winstons_donut_logs
  • Populating the table with time-series data tracking donut consumption

    • The final product is a clean, well-structured database that tracks the evolution of Winston’s donut eating behavior over 31 life stages, incorporating age, context, and life status.

    Technologies Used

    • Database: MySQL
    • Client Tool: MySQL Workbench or CLI
    • Language: SQL

      • Table Schema

      The table winstons_donut_logs includes the following fields:

      • id – Primary key (INT)
      • status – Life stage or status of Winston (VARCHAR)
      • years_old – Age at the time of record (INT)
      • donuts_eaten – Number of donuts consumed (INT)
      • reason – Contextual details explaining the consumption behavior (TEXT)

        • Implementation Overview

        • Database Initialization: 
          CREATE DATABASE DataDig;
USE DataDig;

        • Table Creation: 
          CREATE TABLE winstons_donut_logs (
  id INT PRIMARY KEY,
  status VARCHAR(255),
  years_old INT,
  donuts_eaten INT,
  reason TEXT
);

        • Data Insertion:

          31 rows of data were inserted to reflect Winston's donut consumption at each life stage. 

          INSERT INTO winstons_donut_logs (id, status, years_old, donuts_eaten, reason)
VALUES (1, 'Toddler', 3, 0, 'Too young for donuts'),
       (2, 'Kindergartener', 5, 1, 'Tried donut for the first time'),
       ...
       (31, 'Senior Citizen', 70, 2, 'Still enjoying life one donut at a time');

          (Note: The full script includes all 31 rows in the actual SQL file.)

          • Analytical Value

          While humorous in context, this project mirrors real-world data logging use cases in behavioral analysis and consumption tracking. It teaches fundamental skills like:

          • Database schema design and planning
          • SQL syntax and execution flow
          • Inserting, organizing, and maintaining structured records

            • Getting Started

            • Ensure MySQL is installed and running on your machine
            • Clone this project’s repository or copy the SQL script
            • Open MySQL Workbench or terminal
            • Execute the script step-by-step:
              • Start MySQL: mysql.server start
              • Login: mysql -u root
              • Create and use database: CREATE DATABASE DataDig; USE DataDig;
              • Create the table and insert data

                • Extension Possibilities

                • Use SELECT queries to analyze donut consumption trends
                • Visualize donut consumption by age or status using a dashboard tool
                • Apply window functions to track cumulative totals or running averages

                  • Key Takeaways

                  • Even the simplest data can be transformed into structured, meaningful insights
                  • Hands-on SQL projects build a solid foundation for querying and reporting
                  • Well-documented schemas and scripts improve maintainability and collaboration
